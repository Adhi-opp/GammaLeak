2. Per-symbol row (main table)

One row per tracked symbol. Read left-to-right: price → deviation → regime → flow → signal.

2.1 Price & deviation

TermRange / unitMeaning & use
LTP₹ priceLast traded price from the live tick.
Chg%%Intraday change vs prev close.
VWAP₹ priceVolume-weighted avg price since open. Price above VWAP = intraday bulls in control; below = bears.
Z−5 … +5 (capped)How many SDs price is from its short-window mean. |Z|≥2 = stretched → fade candidate. |Z|≥3 = extreme. Z is capped so outliers don't distort.
SD₹Rolling standard deviation with an ATR-based floor (prevents Z from exploding in quiet sessions). If SD ever looked "stuck high," that's the floor biting.

2.2 Regime indicators

TermRangeMeaning
ER0 … 1Efficiency Ratio. |net move| / sum|per-bar moves|. High (>0.5) = clean trend, don't fade. Low (<0.3) = chop/mean-revert, fades work.
HR0 … 1Hurst exponent. Memory / persistence of returns. >0.55 = trending; <0.45 = mean-reverting; ≈0.5 = random walk. Pair with ER — both low is the cleanest fade setup.
DriftsignedShort-horizon directional bias. Used as a fade guard: a fade-long against strong negative drift is a bad trade — GammaLeak now inverts correctly (prior bug fixed).
Fade-quality checklist: |Z|≥2 AND ER<0.3 AND HR<0.45 AND drift aligned with fade direction. Those four together = high-conviction fade. Miss one → drop a star.

2.3 Options flow (per underlying)

TermRangeMeaning
PCR0.4 … 2.0Put-Call OI Ratio (ATM±10 strikes). >1.3 = put-heavy (support forming → contrarian bullish). <0.7 = call-heavy (supply → contrarian bearish). Extreme PCR at pin = expiry risk.
ΔOI / OI flow± lakh contractsNet change in total OI this window. +CE / −PE = writers selling calls (bearish/cap). +PE / −CE = writers selling puts (bullish/floor). Both rising = straddle build (event pricing).

2.5 Conviction stars (1–5 ★)

Displayed on State-2 signals only. Each point comes from:

  • +1 if |Z| ≥ 2.5 (stretch)
  • +1 if ER < 0.3 (chop)
  • +1 if HR < 0.45 (mean-revert)
  • +1 if drift aligns with the fade direction
  • +1 if OI flow aligns (writers defending the level you're fading)

Default tier policy: trade ★★★★+ on NIFTY weekly, ★★★+ on liquid stocks, skip ★★ and below in real money until stats prove otherwise.

3. Index Drivers panel (Phase 2)

Three pair cards: NIFTY↔BN, NIFTY↔RELIANCE, BN↔HDFCBANK. Refreshes every 15s.

TermRangeMeaning
ρ (rho)−1 … +1Pearson correlation of 1-second log returns over the last 5 min. ≥ 0.7 moving together tightly, 0.4–0.7 loose, < 0.4 decoupled.
lead-lag−30s … +30sGrid search, 5s steps. Negative = symbol A leads B; positive = B leads A. Use it to pick the leading leg for entries.
DRAGchipIndex is lagging its heavyweight — move is narrow, not broad. Fading the index against the heavyweight is risky; the index will eventually catch up.
BOOSTchipIndex and heavyweight move in same direction, index stronger. Broad participation — trend is real.
DIVERGEchipOpposite-sign Z. Low ρ + opposing moves. A heavyweight mean-revert trade; don't apply it to the index.
Policy (Phase 2): this panel is display-only. It does not gate signals yet. If you see DRAG on NIFTY↔RELIANCE and a NIFTY fade-long signal fires, weigh skipping — this is the manual discretionary gate until Phase 4 promotes it.

4. OI Levels — Max-Pain & Gamma Walls (Phase 3, NIFTY only)

Refreshes every 30s. Piggybacks on the 21-strike NIFTY chain (ATM±10, 50-pt step). BN has no chain subscription → no BN card by design.

4.1 Max-Pain

The strike that minimizes total writer payout. Formula:

pain(P) = Σ max(0, P−K)·OI_CE(K) + Σ max(0, K−P)·OI_PE(K)

P* is the gravity price market makers want spot to land on at expiry. Distance shown as dist_pct = (spot − max-pain) / spot.

Use: on expiry days and last 2 hours, expect price to drift toward max-pain unless a real news catalyst overrides. Fade moves away from it.

4.2 Gamma Walls

CE wall (R1/R2/R3)Call-OI cluster above spot. Writers defend it → acts as resistance. Rendered red.
PE wall (S1/S2/S3)Put-OI cluster below spot. Writers defend it → acts as support. Rendered green.
BandWalls filtered to ±3% of spot (far-OTM excluded). Top 3 per side by absolute OI.

4.3 Stale flag

Panel still renders with an explicit reason so the UI never goes blank:

  • no spot — WebSocket hasn't delivered a spot tick yet
  • only N strikes — <5 strikes have non-zero OI (boot warmup)
  • no OI update yet — chain keys subscribed but first OI tick hasn't arrived
  • OI Ns old — last OI update > 120s (connection issue, check logs)

5. How to use the whole screen in 10 seconds

  1. Check BLACKOUT chips first. If active on your symbol, stand down.
  2. Read drivers. DRAG = don't fade the index.
  3. Find a State-2 card with a setup label + ≥3 stars.
  4. Cross-check OI Levels. Fade-long near PE wall (S1) = good. Fade-long into CE wall (R1) = bad, wall will reject.
  5. Check max-pain distance. On Thu afternoons, respect pin gravity — fades toward max-pain have extra tailwind.
  6. Sizing is yours. The tool does not execute.

6. Known limitations (be honest with yourself)

  • BANKNIFTY has no option chain subscribed yet → no max-pain/walls for BN. Phase 3.5 queued.
  • Driver lead-lag grid is 5-second resolution; sub-5s lead shows as 0.
  • Phase 1–3 have not been live-validated at full scale yet — Monday 2026-04-20 is the first real session. Treat signals skeptically until accuracy stats confirm.
  • Conviction score weights are uniform (1 point each). If certain inputs prove stronger post-Monday, weights will be rebalanced.
